# Requisitos

## Requisitos Funcionais

### RF01 – Autenticação e Autorização

| ID    | Descrição do Requisito                                                                                                                           | Prioridade |
|-------|--------------------------------------------------------------------------------------------------------------------------------------------------|------------|
| RF01  | O sistema deve permitir o cadastro de usuário com os campos obrigatórios: Nome completo, E-mail, Senha (com mín. 8 caracteres), Confirmar senha. | Alta |
| RF02  | O sistema deve validar se o e-mail já está cadastrado. Caso esteja, exibir a mensagem: **"Este e-mail já está em uso."**                         | Alta |
| RF03  | A senha deve conter no mínimo 8 caracteres. Caso contrário, exibir: **"A senha deve conter no mínimo 8 caracteres."**                            | Alta |
| RF04  | Caso as senhas não coincidam, o sistema deve exibir: **"As senhas não coincidem."**                                                              | Alta |
| RF05  | Após cadastro realizado com sucesso, o sistema deve exibir: **"Cadastro realizado com sucesso!"** e redirecionar para o dashboard.               | Alta |
| RF06  | O sistema deve permitir login com e-mail e senha.                                                                                                | Alta |
| RF07  | Caso o login seja inválido, o sistema deve exibir: **"E-mail ou senha incorretos."**                                                             | Alta |
| RF08  | O sistema deve permitir recuperação de senha via envio de link para o e-mail cadastrado.                                                         | Média |
| RF09  | Após solicitação de recuperação, exibir: **"Enviamos um link de recuperação para seu e-mail."**                                                  | Média |

---

### RF02 – Cadastro de Veículos

| ID    | Descrição do Requisito                                                                                                                                            | Prioridade |
|-------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------|
| RF10  | O sistema deve permitir cadastro de veículos com os campos: Placa, Tipo (UTI Móvel, Ambulância, 4x4), Ano e Situação (Disponível, Em Atendimento, Em Manutenção). | Alta |
| RF11  | O sistema deve impedir cadastro de placa duplicada. Caso ocorra, exibir: **"Veículo já cadastrado."**                                                             | Alta |
| RF12  | Após cadastro com sucesso, o sistema deve exibir: **"Veículo cadastrado com sucesso."**                                                                           | Alta |
| RF13  | O sistema deve permitir edição dos dados do veículo.                                                                                                              | Alta |
| RF14  | Ao solicitar exclusão, o sistema deve exibir confirmação: **"Tem certeza que deseja excluir este veículo?"**                                                      | Alta |
| RF15  | Após exclusão confirmada, o sistema deve exibir: **"Veículo removido com sucesso."**                                                                              | Média |

---

### RF03 – Cadastro de Funcionários

| ID    | Descrição do Requisito                                                                                                                                         | Prioridade |
|-------|----------------------------------------------------------------------------------------------------------------------------------------------------------------|------------|
| RF16  | O sistema deve permitir cadastro de funcionário com os campos: Nome, CPF, Cargo (Médico, Enfermeiro, Condutor, Socorrista), Telefone e Status (Ativo/Inativo). | Alta |
| RF17  | O sistema deve impedir cadastro de CPF duplicado. Caso ocorra, exibir: **"Funcionário já cadastrado."**                                                        | Alta |
| RF18  | Após cadastro com sucesso, o sistema deve exibir: **"Funcionário cadastrado com sucesso."**                                                                    | Alta |
| RF19  | O sistema deve permitir vincular funcionários a plantões específicos.                                                                                          | Alta |

---

### RF04 – Gestão de Plantões

| ID   | Descrição do Requisito                                                                                                                                | Prioridade |
|------|-------------------------------------------------------------------------------------------------------------------------------------------------------|------------|
| RF20 | O sistema deve permitir cadastro de plantão com: Data, Horário de Início, Horário de Fim, Veículo designado e Equipe designada.                       | Alta |
| RF21 | O sistema deve impedir conflito de horário para o mesmo veículo. Caso ocorra, exibir: **"Este veículo já possui plantão neste horário."**             | Alta |
| RF22 | O sistema deve impedir que um funcionário seja alocado em dois plantões simultâneos. Caso ocorra, exibir: **"Funcionário já alocado neste horário."** | Alta |
| RF23 | Após registro do plantão, o sistema deve exibir: **"Plantão registrado com sucesso."**                                                                | Alta |

---

## RF05 – Gestão de Atendimentos

| ID   | Descrição do Requisito                                                                                                                                                                                                                                                | Prioridade |
|------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------|
| RF24 | O sistema deve permitir o cadastro de serviço/atendimento com os seguintes campos: Tipo de Serviço (ENUM: EVENTO, CHAMADO_EMERGENCIAL, REMOCAO, COBERTURA_CONTRATUAL), Cliente, Data, Horário de Início, Horário de Fim, Local, Veículo designado e Equipe designada. | Alta |
| RF25 | O sistema deve validar disponibilidade do veículo no período selecionado. Caso indisponível, exibir: **"Nenhum veículo disponível para este período."**                                                                                                               | Alta |
| RF26 | O sistema deve validar disponibilidade da equipe no período selecionado. Caso indisponível, exibir: **"Equipe já alocada neste horário."**                                                                                                                            | Alta |
| RF27 | Após cadastro com sucesso, exibir: **"Serviço registrado com sucesso."**                                                                                                                                                                                              | Alta |
| RF28 | O sistema deve permitir alteração do status do serviço (Agendado, Em Andamento, Finalizado, Cancelado).                                                                                                                                                               | Alta |
| RF29 | Caso o usuário tente finalizar um serviço sem equipe vinculada, exibir: **"Não é possível finalizar o serviço sem equipe designada."**                                                                                                                                | Média |

---

### RF06 – Cadastro de Clientes

| ID   | Descrição do Requisito                                                                                   | Prioridade |
|------|----------------------------------------------------------------------------------------------------------|------------|
| RF30 | O sistema deve permitir cadastro de cliente com: Nome, CNPJ/CPF, Telefone, E-mail e Endereço e descrição | Alta |
| RF31 | O sistema deve impedir cadastro duplicado de CNPJ/CPF. Caso ocorra, exibir: **"Cliente já cadastrado."** | Alta |
| RF32 | Após cadastro com sucesso, exibir: **"Cliente cadastrado com sucesso."**                                 | Alta |

---

### RF07 – Dashboard e Relatórios

| ID   | Descrição do Requisito | Prioridade |
|------|------------------------|------------|
| RF33 | O sistema deve exibir no dashboard: veículos disponíveis, plantões ativos, eventos futuros e atendimentos do mês. | Alta |
| RF34 | Caso não haja dados para exibição, mostrar a mensagem: **"Nenhuma informação encontrada."** | Média |
| RF35 | O sistema deve permitir geração de relatório em PDF. | Média |
| RF36 | Após geração do relatório, exibir: **"Relatório gerado com sucesso."** | Média |

---

### RF08 – Controle de Permissões

| ID   | Descrição do Requisito | Prioridade |
|------|------------------------|------------|
| RF37 | O sistema deve possuir perfis de acesso: Administrador e Operador. | Alta |
| RF38 | Apenas usuários Administrador podem excluir registros. | Alta |
| RF39 | Caso um usuário sem permissão tente executar ação restrita, exibir: **"Você não possui permissão para esta ação."** | Alta |

---

## Requisitos Não Funcionais

| ID    | Descrição do Requisito                                                       |
|-------|------------------------------------------------------------------------------|
| RNF01 | O sistema deve responder em até 3 segundos.                                  |
| RNF02 | O tempo de autenticação (login) não deve ultrapassar 5 segundos.             |
| RNF03 | As senhas devem ser armazenadas utilizando criptografia segura.              |
| RNF05 | A sessão do usuário deve expirar após 30 minutos de inatividade.             |
| RNF06 | O sistema deve possuir interface responsiva para desktop e aparelhos móveis. |
| RNF07 | Campos obrigatórios devem estar identificados com "*".                       |
| RNF08 | Mensagens de erro devem ser claras, específicas e objetivas.
| RNF10 | O sistema deve realizar backup automático diário dos dados. |
| RNF11 | O sistema deve operar no modelo multi-tenant, permitindo múltiplas empresas cadastradas. |
